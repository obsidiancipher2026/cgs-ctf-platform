JWT Authentication Bypass

Exploit a weak JWT signing key to forge an admin token.

Medium 105 points

Challenge

The admin panel uses JWT for authentication with a weak, guessable secret key secret123. Forge a JWT with role: admin to access the protected panel and retrieve the flag.

Your guest token is embedded in the page URL. Decode it, modify the role to "admin", re-sign it, and pass it back via the URL.

Instructions

  1. Copy the token from the URL bar
  2. Decode the payload — change "role":"user" to "role":"admin"
  3. Re-sign with the secret key: secret123 using alg: HS256
  4. Replace the token in the URL: ?token=<your_forged_jwt> and press Enter
  5. If the token is valid with role: admin, the flag will appear
Hint: The server uses HMAC-SHA256 with secret secret123. Use an online tool or jwt.io to forge your token.
