## Per-Challenge Details

### Challenge 52 - RSA Small Mistake
- **File**: `rsa_params.txt`
- **Vulnerability**: p and q are too close together (Fermat factorization)
- **Key**: e = 65537
- **n**: 181-digit composite that factors in < 1 second
- **Solver**: Factor n using Fermat's method, compute phi, decrypt

### Challenge 53 - Frequency Fumble
- **File**: `ciphertext.txt`
- **Key**: "LEMUR" (5 characters, revealed to organizers only)
- **Plaintext**: 613 characters of English text containing the flag
- **Solution steps**:
  1. Kasiski examination reveals key length 5
  2. Index of Coincidence confirms key length
  3. Frequency analysis recovers each key letter
  4. Decrypt and extract flag

### Challenge 54 - Penguin Problem
- **Files**: `penguin_original.bmp`, `penguin_encrypted.bmp`
- **ECB Key**: "S3cur3K3yF0rBMP!" (16 bytes)
- **Image**: 320x160 24-bit BMP with flag rendered as horizontal color bands
- **Solution**: Visually compare images; ECB preserves pattern structure
- **Note**: The flag text appears as horizontal bands in both images

### Challenge 56 - Same Prefix Different Story
- **Files**: `file_a.txt`, `file_b.txt`
- **MD5 collision**: Both files have identical MD5 hash `ab787cde71103dc2eaf8574309930669`
- **Technique**: Wang et al collision blocks (differ in 6 byte positions)
- **Verification**: MD5 matches, SHA256 differs, file contents differ
- **Flag**: In the common suffix of both files

### Challenge 57 - Predictable Dice
- **Files**: `token.txt`, `flag_locked.txt`
- **Vulnerability**: Python random seeded with Unix timestamp
- **Timestamp**: 1718352000
- **Solution**: Reproduce random.seed(timestamp), regenerate token, XOR-decrypt flag_locked.txt

### Challenge 58 - Secret's Not So Secret
- **Files**: `token.jwt`, `raw_token.txt`
- **Algorithm**: HS256
- **Weak secret**: "secret123"
- **Payload**: `{"user":"guest","admin":false,"iat":1718352000}`
- **Solution**: Crack secret with hashcat, forge admin=true token

### Challenge 59 - Mode Matters
- **Files**: `ciphertexts.txt`
- **Mode**: AES-CTR (same nonce, same key)
- **Attack**: CTR nonce reuse -> C1 XOR C2 = P1 XOR P2
- **Vulnerability**: Known plaintext attack recovers second message
- **Nonce**: 0x4a6f686e20446f652048656c6c6f2121 ("John Doe Hello!!")
- **Solution**: XOR ciphertexts, XOR with known plaintext 1 to recover plaintext 2 with flag
